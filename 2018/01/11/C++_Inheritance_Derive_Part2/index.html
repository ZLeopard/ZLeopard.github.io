派生类的一般构造函数与析构函数

　　对于派生类的对象来说，它包含一个基类的对象，在执行派生类构造函数之前，先执行基类的构造函数。
在派生类进行初始化时，语法格式为：
　　构造函数(形参表):基类名(基类构造函数实参表){}
注意：不可以将基类构造函数放到派生类构造函数的函数体中，这类似于委托构造函数，只能在初始化列表中执行基类构造函数。
调用方式分为显式调用和隐式调用。
　　显式调用：在派生类的构造函数中，基类的构造函数提供参数。如Bug(int legs, int color):nLegs(legs), nColor(color)
　　隐式调用：若在派生类的构造函数中，省略基类构造函数时，那么将会自动调用基类默认的构造函数。
而派生类的析构函数被执行时，先执行完派生类的析构函数后，自动调用基类的析构函数。可以从下面的代码示例中得到这种结论。

包含成员对象的派生类的构造函数和析构函数

　　在创建派生类对象时，执行派生类的构造函数。首先调用基类的构造函数，再调用成员对象类的构造函数。在执行完派生类的析构函数后，首先调用成员对象类的析构函数，然后调用基类的析构函数。
　　总之，析构函数的调用顺序与构造函数的调用顺序是相反的。

public继承的赋值兼容规则（重点）

　class base{ };
　class derived:public basse{ };
　base b;
　derived d;
1、派生类的对象可以赋值给基类对象（因为派生类相当于是一个基类）
　　b = d; // 反过来就是无效的
2、派生类的对象可以初始化基类引用
　　base & br = d;
3、派生类对象的地址可以赋值给基类指针
　　base * pb = &d;
注意：当继承为protected或private时，这种规则不适用。

多重继承

　　C++中根据基类的个数分为单继承和多重继承，但是多重继承和单继承的基类和派生类的初始化问题十分类似。
在C++中声明多继承的格式为：
　　class <派生类名>:<派生方式1><基类名1>,……<派生方式n><基类名n>{ };
注意：
　　1、在多重继承中，各种派生方式对于基类成员在派生类中的访问权限与单继承规则相同。
　　2、在使用多继承时，对基类的访问应无二义性（下篇文章详讲）。
　　3、多重继承构造函数与析构函数的执行顺序与单继承相同，但是，基类之间的执行顺序是严格按照派生类声明时从左到右的顺序来执行的，与它们在定义派生类构造函数中的执行顺序无关。
在示例代码中，若派生类声明改为：
