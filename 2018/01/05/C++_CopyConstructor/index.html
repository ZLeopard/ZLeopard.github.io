再话C++构造函数以及This指针的秘密

字数统计: 1,455字 | 阅读时长 ≈ 6分钟

作者：TeddyZhang

简介


　　本文主要是对上篇构造函数的再补充，以及学习了解This指针。
　　在C++11标准中，添加了委托构造函数，允许用户可以在一个构造函数中去调用另一个构造函数。但注意的是不可以将被调用的构造函数
写到函数体中，而要写在初始化器中。
　　This指针可以看做C++程序在向C程序“翻译”时对于成员函数如何处理的工具，使用this指针指向成员函数所要作用的对象。

构造函数初始化列表

初始化列表是辅助编写构造函数的一种很方便的技术。从而引入了编写构造函数的另一种方式：
　　类名(参数):初始化列表{语句组}
初始值可以使用构造函数的参数列表。如：
　　Point(double new_x, double new_y) : x(new_x), y(new_y){}
就相当于在构造函数内：
　　x = new_x; y = new_y;
注意：在const数据类型的情况下，使用初始化参数列表是唯一的选择，由于声明为const的数据无法进行一般的赋值。如下面的DATE成员函数中的year变量，被声明为const常量。

委托构造函数

C++11支持的新标准，支持委托构造函数，允许构造函数调用同一类的其他构造函数，但该构造函数必须放在构造函数初始化器中,不能是函数体中。
一般语法：
　　类名(参数列表1):类名(参数列表2){语句组}
　　如：Test(int temp_i) : Test(){ … }
该语法用参数列表1声明了一个构造函数，在被调用时，会调用委托构造函数，可选的传递部分或全部参数到参数列表2。委托构造函数是冒号(:)后出现的构造函数，之后在函数体定义的语句组会被执行。

this指针

其作用是指向成员函数所做用的对象。

  • 非静态（no static）成员函数中可以直接使用this来代表指向该函数作用的对象的指针。
  • 静态成员函数不能使用this指针，由于静态成员函数并不具体作用于某个对象！因此再转换成C语言时，参数的个数不变。

其中this最常见的用法是：使用this返回调用该函数的对象的引用，若要返回自身对象的话，使用return *this;
