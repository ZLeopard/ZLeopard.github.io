静态成员、常量成员函数、封闭类以及友元总结

发表于 2018-01-06 | 分类于 SOFTWARE CODING |
字数统计: 2,846字 | 阅读时长 ≈ 12分钟

作者：TeddyZhang

简介


　　本文主要是对类中出现的静态成员、常量成员函数（对象）、封闭类以及友元概念的总结。
　　静态(static)成员分为静态数据成员和静态成员函数。本质上是全局变量和全局函数，但在类中书写，显得与类紧密联系，但并不作用某一对象。
　　封闭类是指这个类中还包含着另一个类的对象。
　　常量对象与常量成员函数保证了一些数据和方法的安全性，可靠性。
　　友元可以提高程序的执行效率，引入了访问类中私有空间的另一种形式，不单纯通过类的成员函数。

静态成员

在说明前加了static关键字的成员。
　　区别：普通的成员变量每个对象有各自的一份，而静态成员一共就一份，为所有对象共享。
普通成员函数必须作用于某个对象，而静态成员函数并不具体作用于某个对象。因此不通过对象进行调用。本质上是全局函数，设置静态成员这种机制的目的是将和某些类紧密相关的全局变量和函数写到类里面，看上去像一个整体，易于维护和理解。
　　分类：主要包括静态数据成员和静态成员函数。
　　作用：主要用于解决同一类的不同对象之间的数据和函数共享的问题，同一类的不同对象的静态成员均共用一个内存空间。

如果使用sizeof关键字，那么结果为4，由于静态内存均被所有对象共享。
访问方式：

  • 类名::成员名
  • 对象名.成员名 （要清楚，这并不代表作用于这个对象！）
  • 指针->成员名
  • 引用.成员名

常量对象、常量成员函数、常引用

常量对象

如果不希望某个对象的值被改变，则定义该对象的时候可以在前面加上const关键字，变为常量对象。

常量成员函数

在类的常量成员函数说明的后面可以加上const关键字，使之成为常量成员函数。在执行期间不应修改其所作用的对象。因此，在常量成员函数中不能修改成员变量的值(静态成员除外),也不能调用同类非常量成员函数（静态成员函数除外）。
在常量对象上不可以执行非常量成员函数，因为非常量成员函数可能会修改对象的值，所以编译器会自动分析为错误语句。

常引用

　　引用前面可以加const关键字，成为常引用。不能通过常引用，修改其引用的变量。由于当对象作为参数传递时，若直接传递整个对象，会降低效率。通常情况下我们会传递对象的引用，但是这样修改形参的值也会导致实参的变化，这是我们所不想看到的。所以在对象引用前加上const,保证形参对象不可以被修改。

友元

　　众所周知，C++控制类对象私有部分的访问，通常是由共有类的方法提供的唯一途径。但有的时候，这种规则过于严格，不利于编程人员的特定编程问题。于是引入了友元的概念，可以提供另外一种的私有空间的访问形式。友元有三种形式：友元函数，友元类，友元成员函数。

友元类

　　友元类的所有成员函数都是另一个类的友元函数，都可以访问另一个类的隐藏信息（private and protected）
但是使用友元类需要注意以下几点：
　　1、友元关系不可以被继承。
　　2、友元关系是单向的，不具有交换性。B是A的友元，但A并不一定是B的友元。
　　3、友元关系不具有传递性。A是B的友元，B是C的友元，但A不一定是C的友元。
