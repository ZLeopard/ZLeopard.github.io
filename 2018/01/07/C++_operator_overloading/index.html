运算符重载

发表于 2018-01-07 | 分类于 SOFTWARE CODING |
字数统计: 2,515字 | 阅读时长 ≈ 11分钟

作者：TeddyZhang

简介


C++预定义表示对数据的运算：
　　+,-,*,/,^,&,~,!,|,=,<<,>>,!=……
只能用于基本数据类型：整型，实型，字符型，逻辑型…..
　　但是在C++中要面对不同类型的数据，这时候就要使用运算符的重载功能，运算符重载就是使用函数重载的方法，已完成某种特定的操作。运算符重载实质是函数重载，在C++中每一个运算符都对应每一个函数，比如表达式“a+b”在编译前被解释为：operator+(a,b)。所以可以看做是特殊的函数重载。

运算符函数的格式为：
　　<返回类型说明符> operator <运算符符号>(<参数表>)
　　{
　　<函数体>
　　}
运算符函数被调用时分为显式调用和隐式调用两种方法。一般隐式更为简洁明了！

如果对以上的程序进一步深入分析，若声明String对象s1、s2，若直接进行s1 = s2；编译也不会报错，但是程序有可能会终止，这就是浅复制。接下来将深入分析他们的区别。
1、浅复制
　　一开始如图s1指向存有“this”的空间，s2指向存有“that”的空间，当s1 = s2;时，并不是进行一个字节一个字节的复制，而只是修改了s1的指向，与s2共同指向了存有“that”的内存。于是造成了原来s1指向的内存成为内存垃圾，同时若这两个同时消亡，则这片内存会被delete两次，这是所不能允许的，会造成程序崩溃。

2、深复制
　　经过上面的分析，我们要使赋值语句可以使s2指向的内容可以复制到s1所指向的内容，于是这样就不会出现上面的问题。所以我们要再次的重载赋值运算符，使之成为我们想要的形式。

最后进行一个小讨论，解决几个问题！！！
　1、赋值运算符函数 operator=返回值可不可以是void？
　　答：不可以，考虑一下a=b=c; 等价于 a.operator=(b.operator=(c));
　2、传递对象时使用String &而不是String，为什么？
　　答：实现运算符重载时，应尽量保留运算符原来的风格。如(a=b)=c;等价于(a.operator=(b)).operator=(c);意味着b赋值给a，c也赋值给a。
　3、上面的string类就没有任何问题了？

左移右移运算符的重载

　　“<<”和”>>”原本为二进制的左移和右移，但是在C++中常常重载为流插入运算符(cout)和流提取运算符(cin)，那么我们首先确定cout >> 5 >> “this”;这条语句是如何进行的。”>>”是一个双目运算符，也就是说operator>>运算符函数中的参数是两个，由于cout是ostream对象，所以cout >> 5的返回值应该也是ostream对象，从而可以输出”this”。这就等价于cout.operator<<(5).operator<<("this");
　　因此，对<<进行重载时，这样定义重载函数：ostream &operator<<(ostream &o, const CStudents &s);
如果想要将一个对象当做整体输出，那就必须对左移运算符(<<)进行重载！
