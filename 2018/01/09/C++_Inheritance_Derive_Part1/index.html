继承与派生Part1--派生类的访问权限问题

发表于 2018-01-09 | 分类于 SOFTWARE CODING |
字数统计: 1,969字 | 阅读时长 ≈ 8分钟

作者：TeddyZhang

简介


　　本文主要介绍继承的概念和实现方式，以及在继承后形成的新类（派生类）中的一些数据访问权限问题。
　　派生类从基类继承，从而拥有基类的所有成员函数和成员变量，所以内存等于基类内存加上派生类新添加的数据变量。

继承的概念

　　定义：再定义一个新的类B时，如果该类与某个已有的类A相似(即B类拥有A的全部特点)，那么就可以把A作为一个基类，而把B作为基类的一个派生类(也成子类)。派生类是通过对基类进行修改和扩充得到的，在派生类中，可以扩充新的成员变量和成员函数。
　　概念：派生类拥有基类的全部成员函数和成员变量，不论private、protected、public。在派生类的各个成员函数中，不能访问基类中的private成员。
　　功能：继承基类的数据成员，增加新的数据成员，也可以采用重写覆盖的形式修改已有的数据成员。
　　分类：针对基类来说，根据基类的个数分为单继承和多继承。
　　语法格式：class 派生类名:继承方式 基类名

　　　　　　　　// 成员声明:类似于普通类
　　　　　　}

总结C++类访问控制(private/protected/public)

　　private:只可以由该类的成员函数和其友元函数访问
　　　　不能被其他任何访问，该类的对象也不可以。
　　protected:可以由该类的成员函数、派生类的成员函数和其友元函数访问
　　　　但不可以被该类的对象访问。
　　public:可以由该类的成员函数、派生类的成员函数、其友元函数以及该类的对象。
但是在进行类的继承后，基类的属性会发生变化！！！
而派生类的访问权限变成了四种：
　　public、private、pretected、不可访问（接下来会有例子进行分析说明）
一般说法为：
　　类的继承方法属性变化，private属性不能够被继承。
　　使用private继承，基类的protected和public属性在派生类中都变为private。
　　使用protected继承，基类的protected和public属性在派生类中都变为protected。
　　使用public继承，基类的protected和public属性在派生类中都不改变。

特别注意：
　　派生类访问protected成员时，只可以在派生类中通过类成员函数或者在类中声明的派生类对象，不可以在类外通过派生类对象直接访问。总之，protected对象可以看做是可继承的私有成员。

派生类的深入分析

　　由继承的概念来说，通过继承得到的新类为派生类，派生类继承了基类的所有数据成员和除构造函数和析构函数之外的成员函数。
　　在C++中，继承方式分为三种：private、public、protected
　　接下来将逐个分析这三种继承方式！

公有继承

　　基类的公有和保护成员的访问属性在派生类中不变，而基类的私有成员在派生类中不可直接访问。

私有继承

　　基类的公有和保护成员都以私有成员身份出现在派生类中，而基类的私有成员在派生类中不可直接访问。
