初识STL库(数据结构与算法)

发表于 2018-01-15 | 分类于 SOFTWARE CODING |
字数统计: 2,372字 | 阅读时长 ≈ 9分钟

作者：TeddyZhang

简介


　　本文主要讲了STL库的组成成分，初步接触了容器与算法，并大致了解了其分类和简单用法。
　　容器相当于数据结构，而算法可以作用于任何一种容器，即任何一种数据类型，实质为函数模板。
　　学习并掌握STL库对泛型程序设计有很大的作用，对以后工作也有很大的帮助。

STL概述

　　原称为标准模板库(Standard Template Library)，就是一些常用数据结构和算法的模板的集合。比如数据结构中的链表、数组、二叉树等，算法如排序、查找等。有了STL，不必再写大多数的标准数据结构与算法，并且可以获得非常高的性能。
　　容器(Container)：可容纳各种数据类型的通用数据结构，属于类模板
　　迭代器(Iterator)：可用于一次存取容器中元素，类似于指针
　　算法(Algorithm)：用来操作容器中的元素的函数模板。比如sort()可用于对vector中的数据进行排序，find()用来搜索一个list中的对象。由于算法本身与它们所操作的数据类型无关，因此可以从简单数组到高度复杂容器的任何数据结构上使用。

容器概述

　　容器可以用于存放各种类型的数据(基本类型的变量、对象等)的数据结构，都是类模板，可以分为三种：
1、顺序容器
　　向量(vector)、双队列(deque)、列表(list)
2、关联容器
　　集合(set)、多重集合(multiset)、映射(map)、多重映射(multimap)
3、容器适配器
　　栈(stack)、队列(queue)、优先队列(priority_queue)

顺序容器

　　顺序容器并非排序的，元素的插入位置同元素的值无关，分为vector、deque、list三种。
1、vector 头文件
　　vector是动态数组，元素在内存连续存放。随机存取任何元素都能在常数时间完成。在尾部增删元素具有较佳的性能(大部分情况是常数时间)。
2、deque 头文件
　　deque是双向队列，元素在内存中连续存放。随机存放任何元素都能在常数时间完成(但次于vector)。在两端增删元素具有较佳的性能(大部分是常数时间)
3、list 头文件
　　list是双向链表，元素在内存中不连续存放。在任何位置增删元素都能在常数时间完成。不支持随机存取。

关联容器

　　元素是排序的，因此插入任何元素，都按相应的排序规则来确定其位置。在查找时有很好的性能，通常以平衡二叉树方式实现，插入和检索的时间都是O(log(N))。
1、set/multiset 头文件
　　set是集合，set中不允许相同元素的存在，而multiset中允许相同元素的存在。
2、map/multimap 头文件
　　map只能存放对象，map与set的不同在于map中存放的元素有且仅有两个成员变量，一个名为first，另一个为second，map根据first值对元素进行从小到大排序，并可以快速地根据first来检索元素。map与multimap的不同之处在于是否允许相同的first值的元素。

容器适配器

1、stack 头文件
　　stack是栈，是项的有限序列，并满足序列中被删除、检索和修改的项只能是最近插入序列的项(栈顶的项)，并满足后进先出的原则。
2、queue 头文件
　　queue是队列，插入只可以在尾部进行，删除、检索和修改只允许从头部进行。满足”先进先出”的原则。
3、priority_queue 头文件
　　priority_queue是优先级队列，最高优先级元素总是第一个出列。

迭代器

　　迭代器其实是一种泛化的指针，它的作用主要是用于指向顺序容器和关联容器中的元素，用法与指针相似，分为const和非const两种，通过迭代器可以读取它指向的元素。通过非const迭代器还能修改其指向的元素。迭代器上可以执行++的操作，以使其指向容器中的下一个元素。

双向迭代器

若p和p1都是双向迭代器，则可对p、p1可进行以下操作：
　　++p, p++ 使p指向容器中的下一个元素
　　—p, p— 使p指向容器中的上一个元素
　　* p 取p所指向的元素
　　p = p1 赋值
　　p == p1 , p != p1 判断是否相等、不等
注意：双向迭代器不支持大于小于的比较运算！！！

随机访问迭代器

若p和p1都是随机访问迭代器，则可对p、p1可以进行以下操作：
　　双向迭代器的所有操作
　　p += i 将p向后移动i个元素
　　p -= i 将p向前移动i个元素
　　p + i 值为：指向p后面的第i个元素的迭代器
　　p - i 值为：指向p前面的第i个元素的迭代器
　　p[i] 值为：p后面的第i个元素的引用
　　p < p1, p <= p1, p > p1, p >= p1

容器上的迭代器

　　容器 | 容器上的迭代器类别
:——— | :————–
　　vector | 随机访问
　　deque | 随机访问
　　list | 双向(不支持随机访问)
　　set/multiset | 双向
　　map/multimap | 双向
　　stack | 不支持迭代器
　　queue 不支持迭代器
　　priority_queue | 不支持迭代器

算法简介

　　算法其实就是一个函数模板，大多数在<algorithm>中定义，在STL库中提供对各种容器均适用的算法，比如查找，排序等。在STL库中区间大多为左闭右开[begin, end)，一般前者是起始元素的迭代器，后者是终止元素的后面的一个元素的迭代器。有的会返回一个迭代器，如find函数。算法可以处理容器，也可以处理数组，数组名其实也是Type *类型，相当于一个迭代器。

find函数

find()的语法格式为：
　　template
　　Init find(Init first, Init last, const T& val);
其中first与last都是容器的迭代器，他们给出了容器中查找区间的范围[first, last)。并使用==运算符判断相等，函数返回值为一个迭代器，如果找到，则该迭代器指向被找到的元素。如果找不到，则该迭代器等于last。

STL中”大”、”小”与”等于”的概念

STL中”从小到大”的概念：
　　1、在关联容器中内部的元素是从小到大的。
　　2、有些算法要求其操作的区间是从小到大的，称为有序区间算法，例如binary_search
　　3、有些算法会对区间进行从小到大排序，称为排序算法，例如sort函数
STL库中等号的概念：
　　1、有时，“x和y相等”等价于“x==y为真”，例如顺序查找find
　　2、有时，“x和y相等”等价于“x小于y和y小于x同时为假”，例如有序区间算法binary_search
