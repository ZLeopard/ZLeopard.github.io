发表于 2017-05-22 | 分类于 DATA COMPUTING
字数统计: 2,202字 | 阅读时长 ≈ 8分钟

作者：TeddyZhang

简介

本文主要介绍使用Python实现通信过程中的信源熵的计算，并对其进行Huffman编码以及计算出编码效率。
哈夫曼编码是一种最常用信源编码，为最佳码，是可变长编码的一种。
这种编码方式于1952年由Huffman提出，依据字符在文件中出现的概率对字符进行唯一编码，并且保证了可变编码的平均码长最短，从而编码效率较高，
被称为最佳码。
开发环境：Python3.6、Anaconda 1.4.3
所用模块：Python-docx、Matplot、Python-xlwt

发表于 2017-03-19 | 分类于 SOFTWARE CODING
字数统计: 2,136字 | 阅读时长 ≈ 9分钟

作者：TeddyZhang

本文简介

本文主要对生成器和迭代器的概念和用法做一个较为清晰的解释，从而使得一些程序得以解释。
比如for循环其实就是一个iterator，不断的执行next()函数，直到出现StopIteration异常就会终止。
range()函数就相当于是一个生成器，自动生成一个列表，由于列表是可迭代对象，故可以与for语句进行使用。
同样的，list/str/tuple/dict均为可迭代对象，与for语句进行配合使用。

生成器（Generator）

通过列表生成式，我们可以直接创建一个列表。但是，受到内存限制，列表容量肯定是有限的。而且，创建一个包含100万个元素的列表，不仅占用很大的存储空间，如果们仅仅需要访问前面几个元素，那后面绝大多数元素占用的空间就白白浪费了！
所以，我们需要一个边循环边计算的机制，成为生成器：generate
