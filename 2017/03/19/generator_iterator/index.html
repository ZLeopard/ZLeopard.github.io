Python3.5之生成器与迭代器

本文简介

本文主要对生成器和迭代器的概念和用法做一个较为清晰的解释，从而使得一些程序得以解释。
比如for循环其实就是一个iterator，不断的执行next()函数，直到出现StopIteration异常就会终止。
range()函数就相当于是一个生成器，自动生成一个列表，由于列表是可迭代对象，故可以与for语句进行使用。
同样的，list/str/tuple/dict均为可迭代对象，与for语句进行配合使用。

生成器（Generator）

通过列表生成式，我们可以直接创建一个列表。但是，受到内存限制，列表容量肯定是有限的。而且，创建一个包含100万个元素的列表，不仅占用很大的存储空间，如果们仅仅需要访问前面几个元素，那后面绝大多数元素占用的空间就白白浪费了！
所以，我们需要一个边循环边计算的机制，成为生成器：generate

而变成generator的函数，在每次调用next()的时候执行，遇到yield语句返回，再次执行时从上次返回的yield语句处继续执行。
而在调用generator时，我们一般也不会next函数，而是直接使用for循环代替。
Python的for循环本质上就是通过不断调用next()函数实现的。

要理解generator的工作原理，它是在for循环的过程中不断计算出下一个元素，并在适当的条件结束for循环。对于函数改成的generator来说，遇到return语句或者执行到函数体最后一行语句，就是结束generator的指令，for循环随之结束。

可迭代对象(Iterable)与迭代器(Iterator)

迭代器用起来很灵巧，你可以迭代不是序列但表现处序列行为的对象，例如字典的键、一个文件的行，等等。当你使用循环迭代一个对象条目时，你几乎分辨不出它是迭代器还是序列。你不必去关注这些，因为python让它像一个序列那样操作。
迭代器的作用如下：

提供了刻扩展的迭代器接口；
对列表迭代带来了性能上的增强；
在字典迭代中性能提升；
创建真正的迭代接口，而不是原来的随即对象访问；
与所有已经存在的用户定义的类以及扩展得模拟序列和映射的对象向后兼容；
迭代非序列集合（例如映射和文件）时，可以创建更简洁可读的代码。

我们已经知道，可以直接作用于for循环的对象主要有以下几种：
1、集合数据类型：如list / tuple / dict / set / str /等;
2、generator，如生成器和带yield的generator function。

为什么list / dict / str 等数据类型不是Iterator ？

这是因为Python的Iterator对象表示的是一个数据流，Iterator对象可以被next()函数调用并不断返回下一个数据，直到没有数据时抛出StopIteration错误。可以把这个数据流看作是一个有序序列，但我们却不能提前知道序列的长度，只能不断通过next()函数实现按需计算下一个数据，所以Iterator的计算是惰性的，只有在需要返回下一个数据时它才会计算。
Iterator甚至可以表示一个无限大的数据流，例如全体自然数。而使用list是永远不可能存储全体自然数的。
Python的for循环本质上就是通过不断调用next()函数实现的。

自定义Iterator类

自己定义Iterator的类,本身也是Iterable
其实iter()函数是传递两个参数的功能而写，所以如果自己定义一个iterator类，需要使用next函数，这个函数每次返回一个值就可以实现迭代了。迭代器会先进行iter(),再进行next()

注意next中必须有控制iterator的结束条件，不然会出现死循环。
